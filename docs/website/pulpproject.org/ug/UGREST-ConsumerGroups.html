Consumer Groups

Consumer groups let you create and perform certain operations on a group of consumers.

List

List all Consumer Groups

method: GET

path: /pulp/api/consumergroups/

permission: READ

success response: 200 - OK

failure response: None

return: List of Consumer Group objects

sample response: 

 [
  {
    "description": null, 
    "_ns": "consumergroups", 
    "key_value_pairs": {}, 
    "consumerids": [], 
    "_id": "group1", 
    "id": "group1"
  }, 
  {
    "description": null, 
    "_ns": "consumergroups", 
    "key_value_pairs": {}, 
    "consumerids": [], 
    "_id": "group2", 
    "id": "group2"
  }
]


Delete

Deletes a Consumer Group

method: DELETE

path: /pulp/api/consumergroups/<group-id>/

permission: DELETE

success response: 200 - OK

failure response:

  • 404 - Not Found if consumer group with given id does not exist


return: true - when consumer group is deleted successfully

Create

Creates a consumer group

method: POST

path: /pulp/api/consumergroups/

parameters:

  • id <str> Consumer Group ID required
  • description <str> Consumer group description optional - defaulted to None
  • consumerids <list> of <str> Consumer ids to be added to consumer group - optional - defaulted to []


permission: CREATE

success response: 201 - Created

failure response:

  • 409 - Conflict when consumer group with the same id already exists


return: <dict> Newly created consumer group object

sample response: 

 {
  "description": "test-group", 
  "_ns": "consumergroups", 
  "key_value_pairs": {}, 
  "consumerids": ["test-consumer1", "test-consumer2"], 
  "_id": "test-group", 
  "id": "test-group"
}

Add consumer to a group

Adds a consumer to an existing consumer group

method: POST

path: /pulp/api/consumergroups/<group-id>/add_consumer/

parameter: <str> consumer id

permission: EXECUTE

success response: 200 - OK

failure response:

  • 404 - Not Found if a consumer with given id does not exist


return: true if consumer is successfully added to the consumer group

Add key-value to a consumer group

Adds key-value information to an existing consumer group

method: POST

path: /pulp/apiconsumergroups/<group-id>/add_key_value_pair/

parameters:

  • key <str> key - required
  • value <str> value - required
  • force <str> optional, 'true' - force change consumer's key-values in case of conflicting values, else 'false'


permission: EXECUTE

success response: 200 - OK

failure response:

  • 404 - Not Found if consumer group with given id does not exist


return: true if key-values are successfully added to the consumer group

return: true if key-value is successfully deleted from the consumer group

Install Packages

Request that package(s) be installed on a consumer group.

method: POST

path: /pulp/api/consumergroups/<group-id>/installpackages/

parameters:

  • packagenames <list> a list of package names
  • scheduled_time <str> (optional) schedule in iso8601 format; immediate when Null


permission: EXECUTE

success response: 200 - OK

failure response: None

return: <dict> of tasks and job id to check status, 'null' in case of failure to install packages

sample response: 

 {
  "tasks": [], 
  "id": "c48c1467-841f-41b6-a1d5-2dbd3c638ef9"
}
