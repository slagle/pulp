Associate Packages

Associate a collection of filename and checksum tuples to a list of repositories.

method: POST

path: /services/assocate/packages/

permission: EXECUTE

success response: 200 - Ok

failure response:

  • None

{
  "package_info": [

      ], 
      [
        "repo_id"
      ]
    ]
  ]
}

{}

{

    }
}

Initiate File Upload

Initiate a file upload to the Pulp server.
A file upload to Pulp consists of 3 steps:

1 Initiate the file upload to tell Pulp metadata about the file.
1 Append data to the initiate file. This call uploads the actual file data.
1 Import file to finialize the upload and tell Pulp all the data has been uploaded.


method: POST

path: /upload

parameters:

  • name <str> filename
  • checksum <str> file checksum
  • size <str> file size


permission: EXECUTE

success response: 200 OK

failure response:

  • None


return: file id and offset of where to begin upload append
sample request: 

{
    "name": "/tmp/post.txt",
    "checksum": "c270cc7da31eb20e2cc9ba42d971d6ec16441009",
    "size": "19"
}


sample response: 

{
    "id": "cf033af1-7be2-4150-89ce-12fd14da65a1", 
    "offset": 0
}

Append File Upload

Append to a file upload to the Pulp server.
A file upload to Pulp consists of 3 steps:

1 Initiate the file upload to tell Pulp metadata about the file.
1 Append data to the initiate file. This call uploads the actual file data.
1 Import file to finialize the upload and tell Pulp all the data has been uploaded.


method: PUT

path: /upload/append/<file-id>>

parameters:

  • <file-id> is the id returned from the initiate file upload call
  • file data is the body of the request


permission: EXECUTE

success response: 200 OK

failure response:

  • None


return: True if data was successfully appended to the file

sample response: 

true

Import File Upload

A file upload to Pulp consists of 3 steps:

1 Initiate the file upload to tell Pulp metadata about the file.
1 Append data to the initiate file. This call uploads the actual file data.
1 Import file to finialize the upload and tell Pulp all the data has been uploaded.


method: POST

path: /upload/import

parameters:

  • uploadid <str> file id returned during the initiate call and used during the append call
  • metadata <dict> consisting of:
    • type <str> type of file, either rpm or file
    • checksum <str> package checksum
    • hashtype <str> package checksum algorithm
    • pkgname <str> package name
    • nvrea <str> package nvrea
    • size <str> package size
    • description <str> package description
    • license <str> package license
    • group <str> package group
    • vendor <str> package vendor
    • requires <str> package requires


permission: EXECUTE

success response: 200 OK

Agent Status

Get the availability of an agent.

method: POST

path: /agent/status

parameters:

  • filter : <list> of agent uuids


permission: READ

success response: 200 OK

failure response:

  • None


return: <dict>:

  • id <str> the agent id
  • status <dict> agent status:
    • online <bool> indicates agent is online based on heartbeat
    • heartbeat <str> date/time of last heartbeat received
    • capabilities <dict> agent capabilities:
      • bind <bool> (optional) agent supports bind/unbind API
      • heartbeat <bool> (optional) agent supports (sends) heartbeats



sample response: 

{
    'f15': {
        'online' : true,
        'heartbeat' : "2012-01-04 19:44:38.684277+00:00",
        'capabilities' :
             { 'bind' : true,
               'heartbeat' : true
             }
    },
}

Repository Content Export

Export the repository's content into target directory from its source.

method: POST

path: /services/export/repository/

permission: EXECUTE

success response: 202 Accepted

failure response:

404 Not Found if the id does not match a repository
406 Not Acceptable if the repository does not have a source
409 Conflict if a export is already in progress for the repository


return: a Task object

parameters:

  • repoid <str> repository to export
  • target_location <str> target location on the server filesystem where the content needs to be exported
  • generate_isos (optional) <boolean> wrap the exported content into iso image files.
  • overwrite (optional) <boolean> overwrite the content in target location if not empty
