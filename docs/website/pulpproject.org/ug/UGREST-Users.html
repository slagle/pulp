Pulp

Users

Pulp includes a user model to allow access control to specific API methods and the objects stored within Pulp. The goal is to allow you to specify specific users the right or denial to do things such as:

  • create repos
  • modify existing repos
  • add packages to repos

Eventually all API methods with have a specification on which level of access is required to call them.

Base URL Path: /users/

List All Users

List all users

method: GET

path: /users/

parameters: None

permission: READ

success response: 200 - OK

failure response: N/A

return: list of: <dict> User object

sample response: 

[{'id': 'fb12d975-1f33-4b34-8ac9-0adb6089fb87',
  'login': 'tito',
  'name': 'Tito Jones',
  'roles': [u'consumer-users']},
{'id': '10df8a48-e4ec-474d-85c5-a7f657870cb3',
  'login': 'admin',
  'name': 'Admin User',
  'roles': [u'consumer-users']}
]

Create

Creates a new user in the Pulp server.

method: POST

path: /users/

parameters:

  • login: required unique login used to identify this user in human readable cases (CLI, webui)
  • password: required password to specify
  • name: required A human readable name to use for display purposes.

permission: CREATE

success response: 201 Created

failure response: 409 Conflict if there is an existing user with the same login

return: <dict> User object

sample response: 

 {
  "_id": "b79c5544-8cd7-4057-a29d-6b7cce252992", 
  "name": null, 
  "roles": [], 
  "_ns": "users", 
  "login": "jortel2", 
  "password": "JxAHnHlp4p0=,hA0fKCNsKYmgGVcS68F7Dr2V6e+8EQqnUYHdqLbSfkE=", 
  "id": "b79c5544-8cd7-4057-a29d-6b7cce252992"
}

Information

Get information on a user, given the user's id

method: GET

path: /users/<user id>/

parameters: None

permission: READ

success response: 200 OK

failure response: 404 Not Found

return: <dict> a User object

sample response: 

{'id': 'fb12d975-1f33-4b34-8ac9-0adb6089fb87',
 'login': 'tito',
 'name': 'Tito Jones',
 'roles' : [u'super-users']}

Delete

Delete a user

method: DELETE

path: /users/<user id>/

parameters: None

permission: DELETE

success response: 200 OK

failure response: 404 Not Found

return: <bool> True (on success)

sample response: 

True
