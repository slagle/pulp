Unregister

Unregisters a Consumer from pulp server

method: DELETE

path: /pulp/api/consumers/<consumer-id>/

permission: DELETE

success response: 200 - OK

failure response:

  • 404 - Not Found if consumer with given id does not exist


return: true - when consumer is deleted successfully

Register

Registers a consumer to pulp server

method: POST

path: /pulp/api/consumers/

parameters:

  • id <str> Consumer ID required
  • description <str> Consumer description optional - defaulted to null
  • key_value_pairs <dict> Key-value information for the consumer optional - defaulted to {}
  • capabilities <dict> Consumer capabilities optional - defaulted to {}
    • heartbeat <bool> The consumer supports sending heartbeats
    • bind <bool> The consumer supports bind/unbind requests


permission: CREATE

success response: 201 - Created

failure response:

  • 409 - Conflict when consumer with the same id already exists


return: <dict> Newly created consumer object

Consumer History

Retrieves history of events that occurred on a consumer

method: POST

path: /pulp/api/consumers/<consumer-id>/history/

parameters:

  • event_type <str> type of events to retrieve - supported types ("consumer_registered", "consumer_unregistered", "repo_bound", "repo_unbound", "package_installed", "package_uninstalled", "errata_installed", "profile_changed")
  • limit <str> maximum number of results to retrieve
  • sort <str> direction of sort by event timestamp; possible values: 'ascending', 'descending'
  • start_date <str> earliest date of events that will be retrieved; format: yyyy-mm-dd
  • end_date <str> latest date of events that will be retrieved; format: yyyy-mm-dd


permission: EXEUTE

success response: 200 - OK

failure response:

  • 404 - Not Found if consumer with given id does not exist


return: <list> of history objects

List applicable errata for consumers from given set of repositories

Retrieves list of errata applicable for consumers in a given set of repositories. Result also returns consumer ids for each erratum.

method: GET

path: /pulp/api/consumers/applicable_errata_in_repos/

parameters:

  • repoids <list> of <str> list of repository ids to look for applicable errata
  • send_only_applicable_errata <str> 'true' if only applicable errata should be returned, else 'false'. default value - 'true'


permission: READ

success response: 200 - OK

failure response: 400 - Bad Request in case of invalid input for send_only_applicable_errata

return: <dict> of objects that are mappings of errata id to errata info and applicable consumerids

sample response: 

{"RHBA-2008:0279": {"consumerids": ['consumer1'], 
                    "repoids": ["abc"], 
                    "type": "bugfix", 
                    "severity": "", 
                    "title": "xen bug fix update"}, 
 "RHBA-2008:0291": {"consumerids": ['consumer1', 'consumer2'], 
                    "repoids": ["abc"], 
                    "type": "bugfix", 
                    "severity": "", 
                    "title": "libvirt bug fix and enhancement update"},
}


Install Package

Request that package(s) be installed on a consumer.

method: POST

path: /consumers/<consumer id>/installpackages/

parameters:

  • packagenames <list> a list of package names
  • scheduled_time <str> (optional) schedule in iso8601 format; immediate when Null

permission: EXECUTE

success response: 202 ACCEPTED

failure response:

  • 404 Not Found if there is no such consumer

return: Task

task result <dict>

  • installed <list> A list of packages actually installed. Could be empty if package(s) already installed.
  • reboot_scheduled <bool> Indicates a reboot was requested and scheduled.

Example: 

{ "installed" : ["zsh-4.3.10-4.1.el6.x86_64"], "reboot_scheduled" : False }

Update Package

Request that package(s) be updated on a consumer.

method: POST

path: /consumers/<consumer id>/updatepackages/

parameters:

  • packagenames <list> an (optional) list of package names; empty=ALL
  • scheduled_time <str> (optional) schedule in iso8601 format; immediate when Null

permission: EXECUTE

success response: 202 ACCEPTED

failure response:

  • 404 Not Found if there is no such consumer

return: Task

task result <dict>

  • updated <list>:
    • <str> a list of packages actually updated
    • updates <list> a list of packages updated
    • obsoletes <list> a list of pacakges obsoleted

Example: 

[
  "zsh-4.3.10-4.1.el6.x86_64", { "updates" : ["zsh-4.1.10-4.1.el6.x86_64"], "obsoletes" : [] },
  "samba-3.5.6-71.fc14.x86_64", { "updates" : ["samba-3.5.4-71.fc14.x86_64"], "obsoletes" : [] }
]
