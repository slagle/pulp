Grant Role Permissions

Grant permissions for a resource to a role

method: POST

path: /permissions/role/grant/

permission: READ

success response: 200 OK

failure response: 400 Bad Request if the required parameters are not present

return: true

parameters:

  • rolename (required) <str> name of role to grant permissions to
  • resource (required) <str> uri path of resource to grant permissions on
  • operations (required) <list of strings> valid operations are: CREATE, READ, UPDATE, DELETE, EXECUTE

Show Permissions

Show the permissions for a given resource

method: POST

path: /permissions/show/

permission: super user only

success response: 200 OK

failure response: 400 Bad Request if the required parameters are not present

return: permissions for given resource

example: 

{"id": "78daa991-ec1f-4908-9b59-5c1010bde7a6",
 "resource": "/",
 "users": {"admin": ["CREATE", "READ", "UPDATE", "DELETE", "EXECUTE"]}
}


parameters:

  • resource (required) <str> resource path
