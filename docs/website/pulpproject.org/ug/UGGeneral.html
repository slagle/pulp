Pulp General Reference

ID Restrictions

For many entities in Pulp, an ID is used to uniquely identify one instance from another. These IDs are later used in CLI commands and URLs. As such, there are some restrictions on their values. The following are valid characters in an ID:

  • Letters
  • Numbers
  • Underscore _
  • Hyphen -
  • Period .

Time Units

Time units, like those used in timeout arguments, are in the  ISO8601 duration format and allow for the following units:

  • weeks [W]
  • days [D]
  • hours [H]
  • minutes [M]
  • seconds [S]

The general format is PnWnDTnHnMnS where n are positive integer values. The required P designates the duration, also known as a period and the T is required only if hour, minute, and/or second fields are supplied to deliminate the date portion from the time portion. 

$ pulp-admin repo sync --id pulp-f14-64 --timeout P1DT12H

Date Units

Dates are specified to both pulp-admin and pulp-consumer using the  ISO8601 combined date and time format. Pulp can parse both the basic and extended formats. It is not required to provide the time portion when only a date if required. For example: 

$ pulp-consumer consumer history --start_date 2010-01-01
