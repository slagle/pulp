Errata

An erratum is an entity in pulp representing update metadata information for a given package update. Errata usually has associations with Packages, Repo and Consumers.

Base URL Path: /pulp/api/errata/

List / List by type

List all available errata

method: GET

path: /pulp/api/errata/

filters: <str> errata type - e.g. security, bugfix, enhancement etc. optional

permission: READ

success response: 200 - OK

Create

Creates a new errata.

method: POST

path: /pulp/api/errata/

parameters:

  • id <str> Errata ID required
  • title <str> human-readable title for erratum
  • description <str> errata description
  • version <str> errata version
  • release <str> errata release
  • type <str> type of errata eg. 'bugzilla', 'enhancement', 'security'
  • status <str> Erratum status information
  • updated <str> updated erratum date
  • issued <str> erratum issued date
  • from_str <str> rwho issued the erratum
  • reboot_suggested <str> if errataum requires a reboot upon applying
  • references <list> of str includes bugzilla id info, CVE info for security errata etc
  • pkglist <list of package objects> packages effected by this erratum
    Example - 
         [
      {"packages": 
       [
         {
          "src": "http://download.fedoraproject.org/pub/fedora/linux/updates/11/x86_64/pulp-test-package-0.3.1-1.fc11.x86_64.rpm",
          "name": "pulp-test-package", 
          "type": "sha256", 
          "sums": "9d05cc3dbdc94150966f66d76488a3ed34811226735e56dc3e7a721de194b42e", 
          "filename": "pulp-test-package-0.3.1-1.fc11.x86_64.rpm",
          "epoch": "0", 
          "version": "0.3.1", 
          "release": "1.fc11", 
          "arch": "x86_64"
         }
       ], 
       "name": "1", 
       "short": ""
      }
     ]


  • immutable <str> can this erratum be modified
  • repo_defined <str> should this errata added as part of a repo vs custom
  • severity <str> erratum severity (Currently supports: [Critical, Important, Moderate, Low]
  • summary <str> erratum summary information
  • solution <str> any solution text associated to the errtum
  • rights <str> rights information'


permission: CREATE

success response: 201 - Created

failure response: None

Delete

Deletes errata from pulp server

method: DELETE

path: /pulp/api/errata/<errata-id>/

permission: DELETE

success response: 200 - OK

failure response: None

return: true - when errata is deleted successfully, else null
